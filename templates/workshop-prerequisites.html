    Machine requirements

    Laptop with any OS with JDK and Maven will suffice, as we will be using Storm Local Mode for our analysis.

    GitHub Repo

    Please check Storm project samples on my GitHub account:

    • https://github.com/P7h/StormWordCount
    • https://github.com/P7h/StormTweetsWordCount

    JDK 7.x setup

    1. Download latest JDK 7.0 from: http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads- 1880260.html
    2. Please select appropriate version depending on your OS and bits [32 -bit or 64-bit] and complete the installation.
      • Please remember the location to which you have installed the JDK.
    3. Environment Variables:
      • Please add a new environment variable JAVA_HOME to the location where you installed JDK.
      • Please update the environment variable PATH to include %JAVA_HOME%/bin or $JAVA_HOME/bin as required by your OS.
    4. Try java -version in a command prompt or shell.

    Maven 3.x setup

    1. Download latest Maven from: http://www.carfab.com/apachesoftware/maven/maven-3/3.0.5/binaries/apache-maven- 3.0.5-bin.zip
    2. Uncompress the above downloaded binary to a location on your machine.
      • Please remember the location to which you have uncompressed the above archive.
    3. Environment Variables:
      • Please add a new environment variable MAVEN_HOME to the location where you installed Maven.
      • Please update the environment variable PATH to include %MAVEN_HOME%/bin or $MAVEN_HOME/bin as required by your OS.
    4. Try maven -version in a command prompt or shell

    Create a new Twitter app

    1. Please check and use the simple steps mentioned in the below URL for creating a new Twitter app.
      http://iag.me/socialmedia/how-to-create-a-twitter-app-in-8-easy-steps/
    2. Once you created the Twitter app, please keep OAuth Settings handy:
      • Consumer Key
      • Consumer Secret
      • OAuth Access Token
      • OAuth Access Token Secret
    3. Please copy these values to the appropriate keys in config.properties [https://github.com/P7h/StormTweetsWordCount/blob/master/src/main/resources/config.properties] file present in StormTweetsWordCount project on your local machine.

    Building and executing Topology for StormWordCount

    1. Change into StormWordCount Project.
      cd StormWordCount
    2. And execute the following command on the shell or command prompt, run one of the following commands:
      mvn clean compile exec:java -Dexec.classpathScope=compile -Dexec.mainClass=org.p7h.storm.offline.wordcount.topology.WordCountTopology
      mvn clean compile package && java -jar target/storm-wordcount-1.0-SNAPSHOT-jar-with-dependencies.jar
    3. For more detailed steps on this project and features and info on the output it generates, please refer to: https://github.com/P7h/StormWordCount#stormwordcount

    Building and executing Topology for StormTweetsWordCount

    1. Change into StormTweetsWordCount Project.
      cd StormTweetsWordCount
    2. And execute the following command on the shell or command prompt, run one of the following commands:
      mvn clean compile exec:java -Dexec.classpathScope=compile -Dexec.mainClass=org.p7h.storm.wordcount.topology.WordCountTopology
      mvn clean compile package && java -jar target/storm-tweets-wordcount-0.1-jar-with-dependencies.jar
    3. For more detailed steps on this project and features and info on the output it generates, please refer to: https://github.com/P7h/StormTweetsWordCount#stormtweetswordcount

    INSTALLATION INSTRUCTIONS

    Step 1 : Installation of R

    R is going to be our primary platform for the workshop. R version 3.0.1 (Good Sport) has been released on 2013​05​16 and you can install the same. However its not mandatory to have the latest version installed. Any version of R ≥ 2.14.0 is fine.

    The installation is fairly simple and well documented on the R project website. Some details are given below.

    1.1 Installation on *nix systems

    • Installing from source
      Follow the steps given at http://www.cran.r​project.org/doc/manuals/R​admin.html
    • Installation using binaries on Debian/Ubuntu systems
      1. Add the appropriate entry in your /etc/hosts/sources.list file for the nearest cran mirror ( which is IIT Madras FTP server for us)
        example :
        deb http://ftp.iitm.ac.in/cran/bin/linux/ubuntu precise/
      2. Refresh using
        sudo​ apt get update
      3. Secure APT : The Ubuntu archives on CRAN are signed with the key of "Michael Rutter " with key ID E084DAB9. To add the key to your system with one command use
        sudo apt​key adv ​keyserver keyserver.ubuntu.com ​recv​keys E084DAB9
      4. Install R using
        sudo apt get install r​base​core
    • Installation using binaries on other linux systems

      The binaries are available at http://cran.r​project.org/bin/linux/

    1.2 Installation on Windows

    • R precompiled binaries for Windows are available at http://www.cran.r​project.org/bin/windows/base/R​3.0.1​win.exe
    • Common questions about R installation on Windows are answered at http://www.cran.r​project.org/bin/windows/base/rw​FAQ.html

    1.3 Installation on OS​X

    • R precompiled binaries for OS​X are available at http://www.cran.r​project.org/bin/windows/base/rw​FAQ.html
    • Common questions about R installation on Mac platform are answered at http://www.cran.r​project.org/bin/macosx/RMacOSX​FAQ.html

    Step 2 : Installation of relevant R packages

    Installation of new packages is simple once we have R up and running. Just use the install.packages() function!

    • Most the R packages do a self check after installation ( “testing if installed package can be loaded” )
    • Issues during installation are highlighted ( “installation of package ‘....’ had non​zero exit status” ) with relevant error messages.

    We need the following packages for our session.

    Package name Package usage
    package tm (must!) Main functions for text mining
    package e1071 (must!) Functions for training Naive Bayes Classifier and other machine learning tasks
    package openNLP, openNLPmodels.en (optional) Functions for natural language processing tasks and models trained for English

    2.1 Installing package for text mining

    • This packages needs R >= 2.14 and methods package (installed by default with R)
    • Install the package by running the command
      install.packages(“tm”)

    2.2 Installing package e1071

    Install the package using

    install.packages(“e1071”)

    2.3 Installing package openNLP and openNLPmodels.en (optional)

    • This package is a collection of tools from the openNLP project ( which is in Java) and hence imports package rJava during the installation.
    • Importantly, your system must have JDK 1.4 or higher installed.
    • Install the package using the following command. (It will also install the dependency rJava)
      install.packages(“openNLP”, “openNLPmodels.en”)

    For any issues faced during installation : write to harshad.saykhedkar AT sokrati DOT com with subject line ‘fifth elephant workshop installation issues’

    INSTALLATION INSTRUCTIONS

    Important:You need to have Linux running on your machine. Either do a Dual Boot or use a VM. Running ElasticSearch inside a Linux VM is not only fine, but highly encouraged. Ubuntu/Fedora VMs are equally good. For this workshop, ES examples can run good on anything above 2GB RAM VM, with 20 GB disk.

    1. Install updated JRE for your platform.
    2. Download ElasticSearch from http://www.elasticsearch.org/download/
    3. In case of deb or rpm,
      • install it using your package manager.
      • Start the service using
        service elasticsearch start
        or
        /etc/init.d/elasticsearch start
    4. In case of zip or tarball,
      • extract it at your preferred location.
      • Open the terminal and navigate to the extraced directory.
      • Run
        ./bin/elasticsearch ­-f
    5. Open your web browser and navigate to http://localhost:9200/
    6. If you're greeted with: 
      {
  ok: true,
  status: 200,
  name: "Andrew Gervais",
  version: {
    number: "0.90.1", snapshot_build: false, lucene_version: "4.3"
  },
  tagline: "You Know, for Search" 
}
      ElasticSearch is installed and running. You're good to go!
